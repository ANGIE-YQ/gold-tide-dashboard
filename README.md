# 黄金潮汐看板 · Gold Tide

> 一个用 AI 帮你判断黄金短期走势的看板。每天自动更新，手机电脑都能看。

**看板地址**：[angie-yq.github.io/gold-tide-dashboard](https://angie-yq.github.io/gold-tide-dashboard)

---

## 这是什么？

简单说：**把18年的黄金价格数据喂给AI，让它学会判断未来1-2周金价会涨还是会跌。**

它不会告诉你"明天一定涨"，而是给出一个**概率**。比如"P(涨)=0.69"意思是：根据历史上类似的情况，有69%的概率未来10天会涨。

## 看板怎么看？（30秒看懂）

打开看板，你会看到这些：

| 位置 | 内容 | 怎么看 |
|------|------|--------|
| **右上角** | 实时金价 | 来自交易所，15秒刷新一次 |
| **中间大字** | P(涨)=0.xxx | 大于0.55=看多，小于0.45=看空，中间=观望 |
| **下方** | 止损/目标/仓位 | 模型建议的买卖价位和仓位比例 |
| **市场状态** | 牛熊/波动/超买超卖 | 判断大盘环境是热是冷 |
| **图表区域** | 潮汐图/动能图 | 蓝色=上涨段，红色=下跌段，看颜色就知道趋势 |

右上角绿灯亮 = 实时价格正常更新。灰灯 = 网络异常，不影响模型数据。

## 它有多准？

模型在历史数据上做的"盲测"（用过去数据预测未来，完全不偷看）：

| 预测周期 | 准确率 | 说明 |
|---------|--------|------|
| 5个交易日 | 61% | 短期信号较强 |
| 10个交易日 | 60% | **主要使用这个** |
| 20个交易日 | 53% | 偏向随机，不可靠 |

> 作为对比：闭眼猜的正确率约 55%（因为金价长期上涨略多于下跌）。模型比闭眼猜高出约 5 个百分点，在金融预测里算不错的水平。

**回测成绩**（模拟真实交易，含手续费）：

| 成本 | 年化收益 | 最大回撤 |
|------|---------|---------|
| 万分之五/次 | +49% | -14% |
| 万分之二十/次 | +35% | -16% |
| 买入不动 | +10% | — |

> 模型在牛市和熊市里表现差不多——说明它靠的不是"赌金价永远涨"，而是真的在判断方向。

## 核心思路（通俗版）

### 把金价想象成海浪

海浪不是一条直线——它有大浪（涨几个月）、有小浪（涨几天）、有回头浪（反弹）。

模型的思路是：
1. **先把历史上每一段涨跌切出来**，大段叫"大潮汐"，大段里的小波动叫"小潮汐"
2. **给每段打分**——涨得干脆利落的分数高，涨得磨磨唧唧的分数低
3. **比较相邻的浪**——如果这次的浪比上次小，可能快要转向了
4. **AI 学习所有浪的特征**，找出哪些特征最容易预示接下来会涨

这样当今天的金价走势呈现出某种特征时，模型就能说："历史上出现过 100 次类似情况，其中 69 次接下来涨了"。

### 为什么不用简单规则？

最开始试过"动能弱了就买"这种简单规则——结果准确率只有 50%，跟抛硬币一样。因为市场太复杂，简单规则覆盖不了所有情况。

所以改成了：**把潮汐特征喂给 AI，让 AI 自己找出规律**。这个改动让准确率从 50% 提升到 60%。

## 诚实地说：它的局限

1. **只预测短期**（1-2周），不适合用来判断"下个月金价会不会创新高"
2. **只看价格形态**，不看新闻、利率、美元——这些因素会影响金价但模型目前没纳入
3. **回测好≠实盘好**——过去的表现不能保证未来，实盘会有滑点、延迟
4. **单次预测可能错**——60%准确率意味着10次里有4次是错的

## 如何更新数据？

### 自动更新（推荐）

把 `setup-workflow.yml` 移动到 `.github/workflows/update-dashboard.yml`（在 GitHub 网页上操作），之后每个交易日收盘后自动刷新。

### 手动更新

```bash
cd D:/Work/module
python refresh_dashboard.py           # 拉数据 + 重训练 + 刷新看板
git add docs/ gold_AU0_daily.csv gold_features.csv
git commit -m "update" && git push    # 推送后网站自动更新
```
